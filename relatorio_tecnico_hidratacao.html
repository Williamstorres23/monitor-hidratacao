1. Apresentação

O presente projeto propõe o desenvolvimento de um sistema embarcado de monitoramento e incentivo à hidratação, utilizando a placa de desenvolvimento BitDogLab baseada no microcontrolador RP2040. O sistema atua como um lembrete inteligente, auxiliando o usuário a manter hábitos saudáveis de consumo de água ao longo do dia.

O problema abordado é a dificuldade que a maioria das pessoas enfrenta em se lembrar de beber água regularmente, especialmente durante atividades de trabalho ou estudo prolongadas. A desidratação provoca redução na concentração, fadiga, dores de cabeça e problemas de saúde a longo prazo. Soluções existentes, como aplicativos de smartphone, exigem que o usuário consulte ativamente o dispositivo — o que frequentemente é esquecido.

A motivação para esta solução é criar um dispositivo dedicado, autônomo e não invasivo, que monitora passivamente o tempo e alerta o usuário com feedbacks sonoros e visuais no momento certo, sem depender de interação contínua. Adicionalmente, o sistema conecta-se à internet para registrar e transmitir dados de hidratação via protocolo MQTT, permitindo acompanhamento por dashboards externos.

2. Objetivos

2.1 Objetivo Geral

Desenvolver um sistema embarcado completo na BitDogLab (RP2040) capaz de monitorar e incentivar a hidratação do usuário por meio de alertas periódicos, registro de consumo e transmissão de dados via protocolo MQTT sobre WiFi.

2.2 Objetivos Específicos

  • Implementar um timer de interrupção configurável para disparar alertas em intervalos regulares de 30 minutos
  • Desenvolver interface visual no display SSD1306 com informações de copos bebidos, volume total e countdown
  • Implementar feedback sonoro diferenciado via buzzer PWM para alertas e confirmações de copo
  • Capturar a interação do usuário via botões com tratamento de interrupções GPIO e debounce por software
  • Estabelecer comunicação WiFi e publicar dados de hidratação via MQTT em formato JSON
  • Estruturar o firmware em módulos coesos e bem documentados seguindo boas práticas de programação embarcada

5. Arquitetura do Firmware

O firmware foi estruturado seguindo o princípio da separação de responsabilidades, com cada módulo sendo responsável por uma única funcionalidade do sistema. Esta abordagem facilita a manutenção, o teste individual dos componentes e a reutilização de código.

5.1 Módulo principal — main.c

Ponto de entrada do sistema. Responsável por inicializar todos os módulos na sequência correta, registrar os callbacks de interrupção e executar o loop principal. O loop verifica flags setadas pelas interrupções e coordena as ações entre os módulos.

5.2 Módulo de temporização — timer_module

Utiliza a função repeating_timer_ms() do Pico SDK para gerar uma interrupção a cada 1 segundo. Ao atingir o intervalo configurado (padrão: 1800 segundos = 30 minutos), o callback do usuário é invocado. O módulo expõe funções para resetar, pausar e retomar a contagem.

5.3 Módulo de display — display_module

Abstrai a comunicação I2C com o display SSD1306 e fornece funções de alto nível para atualizar a interface. O display é dividido em quatro linhas: título do sistema, contador de copos, countdown do próximo lembrete e volume total em mL.

5.4 Módulo de entrada — input_module

Configura os pinos GPIO dos botões com resistores de pull-up internos e registra interrupções na borda de descida (nível lógico baixo). Implementa debounce por software com janela de 200ms para evitar múltiplos acionamentos por um único toque físico.

5.5 Módulo do buzzer — buzzer_module

Gera tons de frequência controlada usando o periférico PWM do RP2040. Oferece dois padrões sonoros distintos: dois beeps longos para alertas de hidratação (1000Hz) e dois beeps curtos e agudos para confirmação de copo registrado (1500Hz).

5.6 Módulo MQTT — mqtt_module

Utiliza a biblioteca lwIP com o cliente MQTT integrado ao Pico SDK para conectar-se a um broker via WiFi (CYW43). Publica um payload JSON contendo o número de copos bebidos, o volume total em mL e o tempo restante em segundos:

{"copos": 3, "volume_ml": 600, "tempo_restante_s": 1247}

8. Conclusão

O projeto Monitor de Hidratação demonstrou com sucesso a aplicação prática dos conceitos estudados durante o programa EmbarcaTech Expansão. O sistema integra múltiplos periféricos da BitDogLab — display gráfico, buzzer, botões e WiFi — em uma solução coesa e funcional que aborda um problema real de saúde.

Os principais resultados alcançados foram: implementação de um sistema de interrupções estável e responsivo, modularização eficiente do firmware em C, comunicação IoT funcional via MQTT e interface de usuário intuitiva no display SSD1306.

Dificuldades Encontradas

  • Configuração da pilha lwIP para uso do cliente MQTT em conjunto com o driver CYW43, que requer cuidados específicos com threading
  • Implementação do debounce por software de forma confiável sem bloquear as interrupções
  • Gerenciamento do tempo de inicialização do WiFi sem travar a interface do usuário

Melhorias Futuras

  • Adicionar sensor de temperatura para sugerir maior ingestão de água em dias quentes
  • Implementar persistência da contagem diária em memória flash (usando LittleFS)
  • Desenvolver dashboard web em Node-RED ou Grafana para visualização histórica dos dados
  • Adicionar configuração do intervalo de alerta via joystick e display, sem necessidade de recompilar
  • Implementar transferência de dados com DMA para otimizar a comunicação I2C com o display (bônus)

Em conclusão, o projeto cumpriu todos os requisitos obrigatórios do EmbarcaTech Expansão, utilizando a plataforma BitDogLab com programação em C/C++, integração de periféricos, comunicação sem fio WiFi e protocolo IoT MQTT, consolidando o aprendizado em sistemas embarcados com conectividade.
